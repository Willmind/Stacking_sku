# 项目输入需求说明

本文档基于当前仓库代码整理，目标是把“智能装柜助手”目前需要用户输入、系统派生输入、批量导入输入、算法层入参和展示控制输入统一列清楚，方便后续产品设计、测试用例、Excel 模板和算法迭代保持一致。

## 1. 输入范围总览

当前项目的输入可以分为 6 类：

| 输入类别 | 是否影响装载计算 | 当前入口 | 主要用途 |
| --- | --- | --- | --- |
| 集装箱规格 | 是 | 左侧「集装箱规格」下拉框 | 决定可装载空间长宽高 |
| 车厢公差 | 是 | 左侧「车厢公差」数字输入 | 扣减有效装载空间，预留防潮纸皮、装柜间隙等空间 |
| 码垛模式 | 是 | 左侧「码垛模式」单选切换 | 决定使用单 SKU 最大装载或多 SKU 目标装载 |
| 纸箱 / SKU 规格与朝向 | 是 | 单 SKU 表单或多 SKU 卡片 | 决定纸箱尺寸、允许朝向、目标数量、SKU 颜色和顺序 |
| 批量导入 Excel | 是 | 「批量导入 Excel」文件选择 | 批量计算多行纸箱的最大装载量和差值 |
| 展示控制 | 否，只影响可视化 | 进度条、2D 视角、3D 交互、放大弹窗 | 控制当前显示多少箱、从哪个视角查看结果 |

当前算法共同前提：

- 尺寸单位默认为 `mm`。
- 单 SKU 和多 SKU 支持纸箱朝向勾选；默认只启用 `长 × 宽 × 高`、`宽 × 长 × 高` 两种水平旋转朝向。
- 开启侧放朝向后，算法可在 `长 × 宽 × 高`、`宽 × 长 × 高`、`长 × 高 × 宽`、`高 × 长 × 宽`、`宽 × 高 × 长`、`高 × 宽 × 长` 六种朝向中按勾选项尝试排布。
- 批量导入 Excel 当前仍使用默认水平旋转朝向，暂未开放每行朝向列。
- 当前支持车厢级公差，用于扣减有效装载空间；不输入承重、重心、纸箱压缩、层间稳定、纸箱公差、装卸通道等约束。
- 集装箱内侧顶部角件固定按 `110 × 110 × 80 mm` 避让，当前页面不开放角件尺寸编辑。

## 2. 集装箱规格输入

### 2.1 页面输入项

| 字段 | 输入方式 | 当前可选值 | 默认值 | 是否必填 | 影响 |
| --- | --- | --- | --- | --- | --- |
| 柜型 | 下拉选择 | `20GP`、`40GP`、`40HQ` | `20GP` | 是 | 决定柜体长宽高 |

### 2.2 预设柜型尺寸

| 柜型 | 长 mm | 宽 mm | 高 mm | 说明 |
| --- | ---: | ---: | ---: | --- |
| `20GP` | `5898` | `2352` | `2393` | 当前默认柜型 |
| `40GP` | `12032` | `2352` | `2393` | 普通 40 尺柜 |
| `40HQ` | `12032` | `2352` | `2698` | 高柜 |

### 2.3 车厢公差输入

车厢公差用于手动预留集装箱内部空间，例如左右铺防潮纸皮、顶部需要留安全余量、柜门或柜内侧需要预留装柜间隙。该输入会直接影响最大装载量。

基准方向：按站在集装箱柜口正视柜内为基准，前为柜子最内部方向，后为柜门方向。

| 字段 | 代码字段 | 输入方式 | 默认值 | 当前约束 | 是否必填 | 影响 |
| --- | --- | --- | ---: | --- | --- | --- |
| 前公差 | `containerClearance.front` | 数字输入 | `0` | 最小 `0`，步进 `1` | 是 | 扣减柜体长度 |
| 后公差 | `containerClearance.rear` | 数字输入 | `0` | 最小 `0`，步进 `1` | 是 | 扣减柜体长度 |
| 左公差 | `containerClearance.left` | 数字输入 | `0` | 最小 `0`，步进 `1` | 是 | 扣减柜体宽度 |
| 右公差 | `containerClearance.right` | 数字输入 | `0` | 最小 `0`，步进 `1` | 是 | 扣减柜体宽度 |
| 顶部公差 | `containerClearance.top` | 数字输入 | `0` | 最小 `0`，步进 `1` | 是 | 扣减柜体高度 |

有效装载空间计算规则：

- 有效长度 = `container.length - front - rear`
- 有效宽度 = `container.width - left - right`
- 有效高度 = `container.height - top`
- 扣减后的有效长度、有效宽度、有效高度都必须大于 `0`。
- 箱体实际坐标会按 `front`、`left` 等偏移回原始柜体坐标系，2D/3D 会显示有效装载空间边界。

持久化与重置：

- 公差配置会保存在浏览器本地存储中，下一次打开页面会沿用上一次设置。
- 点击「重置」会把 `前 / 后 / 左 / 右 / 顶部` 全部恢复为 `0`，并同步更新本地存储。
- 当五项公差全部为 `0` 时，「重置」按钮禁用。

### 2.4 当前边界

- 页面目前只开放预设柜型选择。
- 代码中算法层支持传入自定义 `ContainerSpec`，但页面没有开放自定义柜体长宽高输入。
- 如果未来开放自定义柜型，需要输入：
  - 柜体长度 `container.length`
  - 柜体宽度 `container.width`
  - 柜体高度 `container.height`
  - 三项都必须为正数。

## 3. 码垛模式输入

| 字段 | 输入方式 | 可选值 | 默认值 | 是否必填 | 影响 |
| --- | --- | --- | --- | --- | --- |
| 码垛模式 | 单选切换 | `单 SKU`、`多 SKU` | `单 SKU` | 是 | 决定展示单箱表单还是多 SKU 编辑器 |

### 3.1 单 SKU 模式

用途：计算某一种纸箱在选定柜型中的最大装载量。

特点：

- 不输入目标数量。
- 结果以“最大装载量”为主。
- 2D/3D 中所有纸箱使用同一个箱体颜色。

### 3.2 多 SKU 模式

用途：按 SKU 顺序和目标数量分配装载。

特点：

- 支持 `2` 到 `5` 个 SKU。
- 每个 SKU 有独立尺寸、目标数量和颜色。
- SKU 顺序可以通过拖拽调整。
- 当前异尺寸多 SKU 属于启发式分区算法，不是全局最优混装算法。

## 4. 单 SKU 输入需求

### 4.1 页面字段

| 字段 | 代码字段 | 输入方式 | 默认值 | 当前约束 | 是否必填 | 说明 |
| --- | --- | --- | ---: | --- | --- | --- |
| 纸箱长 | `singleCarton.length` | 数字输入 | `480` | 最小 `1`，步进 `1` | 是 | 单位 `mm` |
| 纸箱宽 | `singleCarton.width` | 数字输入 | `320` | 最小 `1`，步进 `1` | 是 | 单位 `mm` |
| 纸箱高 | `singleCarton.height` | 数字输入 | `260` | 最小 `1`，步进 `1` | 是 | 单位 `mm` |
| 允许朝向 | `singleAllowedOrientations` | 复选项 | `长×宽×高`、`宽×长×高` | 至少选择 1 项 | 是 | 控制单 SKU 可用摆放姿态 |
| 箱体颜色 | `singleColor` | 色板 / HEX / 系统取色器 | `#D8923A` | `#RRGGBB` | 是 | 用于 2D/3D 渲染，不影响装载量 |

### 4.2 朝向输入规则

朝向字段使用 `柜长方向 × 柜宽方向 × 柜高方向` 表达，例如 `长×高×宽` 表示纸箱长度沿柜长、纸箱高度沿柜宽、纸箱宽度沿柜高。

| 朝向 | 代码值 | 当前默认 | 说明 |
| --- | --- | --- | --- |
| `长×宽×高` | `length-width-height` | 是 | 当前默认长向 |
| `宽×长×高` | `width-length-height` | 是 | 当前默认水平 90° 旋转 |
| `长×高×宽` | `length-height-width` | 否 | 侧放，纸箱宽度变为垂直高度 |
| `高×长×宽` | `height-length-width` | 否 | 侧放并在水平面旋转 |
| `宽×高×长` | `width-height-length` | 否 | 侧放，纸箱长度变为垂直高度 |
| `高×宽×长` | `height-width-length` | 否 | 侧放并在水平面旋转 |

约束：

- 至少必须保留 1 种朝向。
- 单 SKU 使用 `singleAllowedOrientations`。
- 多 SKU 每个 SKU 使用自己的 `sku.allowedOrientations`，拖拽排序时朝向配置跟随 SKU 一起移动。
- 算法会在勾选朝向内选择装载结果更优的排布；勾选侧放不等于一定会采用侧放。

### 4.3 颜色输入规则

颜色支持：

- 推荐色板选择。
- 手动输入 HEX 值。
- 浏览器原生颜色选择器。

当前校验规则：

- 支持 `#RRGGBB`。
- 也兼容不带 `#` 的 6 位十六进制字符串，组件会自动补 `#`。
- 非法颜色不会提交。
- 颜色会统一显示为大写，例如 `#D8923A`。

### 4.4 计算触发

用户修改任意单 SKU 字段后，状态会变为：

- 未计算过：`待计算`
- 已有旧结果：`待重新计算`

点击「计算装载」后才会重新计算。

## 5. 多 SKU 输入需求

### 5.1 多 SKU 总体字段

| 字段 | 代码字段 | 输入方式 | 默认值 | 当前约束 | 是否必填 | 说明 |
| --- | --- | --- | ---: | --- | --- | --- |
| SKU 个数 | `skuCount` | 滑杆 | `2` | `2` 到 `5`，会四舍五入并夹紧 | 是 | 控制 SKU 卡片数量 |
| 装载策略 | `strategy` | 下拉选择 | `multi-destination` | 两种预设策略 | 是 | 控制多 SKU 分配逻辑 |

### 5.2 装载策略选项

| 展示文案 | 代码值 | 当前说明 |
| --- | --- | --- |
| 分客户/多卸货地 | `multi-destination` | 按卸货顺序分段，优先保证 SKU 顺序 |
| 同卸货地/完整面优先 | `same-destination` | 优先铺满可视面，使排布更完整 |

### 5.3 SKU 卡片字段

每个 SKU 卡片都包含以下输入：

| 字段 | 代码字段 | 输入方式 | 默认值 | 当前约束 | 是否必填 | 说明 |
| --- | --- | --- | ---: | --- | --- | --- |
| SKU 名称 | `sku.label` | 系统生成 | `A`、`B` 等 | 页面不开放编辑 | 是 | 用于区分 SKU 和渲染标签 |
| 纸箱长 | `sku.length` | 数字输入 | `480` | 最小 `1`，步进 `1` | 是 | 单位 `mm` |
| 纸箱宽 | `sku.width` | 数字输入 | `320` | 最小 `1`，步进 `1` | 是 | 单位 `mm` |
| 纸箱高 | `sku.height` | 数字输入 | `260` | 最小 `1`，步进 `1` | 是 | 单位 `mm` |
| 目标数量 | `sku.target` | 数字输入 | `100` | 最小 `1`，步进 `1`，算法校验为整数 | 是 | 期望装载数量 |
| 允许朝向 | `sku.allowedOrientations` | 复选项 | `长×宽×高`、`宽×长×高` | 至少选择 1 项 | 是 | 控制该 SKU 可用摆放姿态 |
| 箱体颜色 | `sku.color` | 色板 / HEX / 系统取色器 | 按 SKU 预设色 | `#RRGGBB` | 是 | 用于 2D/3D 区分 SKU |

### 5.4 SKU 默认颜色

当前 SKU 颜色按顺序取值：

| SKU 序号 | 默认颜色 |
| --- | --- |
| A | `#D8923A` |
| B | `#42D6A4` |
| C | `#6E8BFF` |
| D | `#FF7066` |
| E | `#B7E35F` |

### 5.5 SKU 顺序输入

SKU 顺序通过拖拽卡片调整：

- 拖拽入口是卡片左侧拖拽按钮。
- 拖拽后的卡片顺序会影响装载顺序。
- 当前 SKU 名称来自对象自身的 `label`，页面不提供手动改名输入。
- 如果未来需要“拖拽后重新命名 A/B/C/D/E”，需要明确 SKU 名称到底代表“商品身份”还是“装载顺序”。

### 5.6 多 SKU 校验规则

算法层当前校验：

- SKU 列表必须是数组。
- SKU 数量必须在 `2` 到 `5` 之间。
- 每个 SKU 必须是对象。
- 每个 SKU 的 `target` 必须为正数。
- 每个 SKU 的 `target` 必须为整数。
- 每个 SKU 的 `length`、`width`、`height` 必须为正数。
- 每个 SKU 至少必须保留 1 种允许朝向。
- SKU 朝向必须来自系统支持的 6 种朝向代码。
- SKU 名称不能重复。
- 颜色为空时，算法层可兜底为 `#d8923a`，但页面正常都会提供颜色。

## 6. 批量导入 Excel 输入需求

### 6.1 文件输入

| 字段 | 输入方式 | 当前约束 | 是否必填 | 说明 |
| --- | --- | --- | --- | --- |
| Excel 文件 | 文件选择 | `accept=".xlsx"` | 是 | 当前页面只接受 `.xlsx` |

### 6.2 Excel 表头

当前批量导入依赖固定表头，表头需要与模板一致：

| 列名 | 示例 | 当前校验 | 说明 |
| --- | --- | --- | --- |
| `人工码垛数量（原始）` | `1470` | 不能为空，必须能转成数字 | 用于和算法最大装载量做差值对比 |
| `尺寸（长宽高 mm）` | `465*360*291` | 不能为空，必须解析出 3 个正数 | 单位默认为 `mm` |
| `柜型` | `40HQ` | 不能为空，只支持 `20GP`、`40GP`、`40HQ` | 会转成大写后匹配 |

### 6.3 尺寸文本格式

尺寸列支持以下分隔符：

| 输入示例 | 是否支持 |
| --- | --- |
| `465*360*291` | 支持 |
| `465x360x291` | 支持 |
| `465X360X291` | 支持 |
| `465×360×291` | 支持 |
| `465-360-291` | 不支持 |

解析规则：

- 文本按 `*`、`x`、`X`、`×` 拆分。
- 拆分后必须正好得到 3 段。
- 三段都必须是有限正数。
- 输出统一格式为 `长*宽*高`。

### 6.4 行处理规则

| 场景 | 当前处理 |
| --- | --- |
| 三个关键字段都为空 | 跳过该行，不生成结果 |
| 尺寸为空 | 生成解析失败结果 |
| 尺寸格式错误 | 生成解析失败结果 |
| 柜型为空 | 生成解析失败结果 |
| 柜型不在 `20GP/40GP/40HQ` | 生成解析失败结果 |
| 人工码垛数量为空 | 生成解析失败结果 |
| 人工码垛数量不能转成数字 | 生成解析失败结果 |
| 算法算出 `totalBoxes > 0` | 状态为 `成功` |
| 算法算出 `totalBoxes <= 0` | 状态为 `无法装载` |

批量导入当前不读取朝向列，计算时按默认 `长×宽×高`、`宽×长×高` 两种水平旋转朝向处理。

### 6.5 批量导入输出字段

导入完成后结果表展示：

| 字段 | 来源 |
| --- | --- |
| 人工码垛数量（原始） | Excel 原始输入 |
| 尺寸（长宽高 mm） | Excel 解析后的尺寸 |
| 柜型 | Excel 解析后的柜型 |
| 最大装载量 | 算法计算结果 |
| 差值 | `最大装载量 - 人工码垛数量（原始）` |
| 状态 | `成功`、`解析失败`、`计算失败`、`无法装载` |
| 失败原因 | 解析或计算失败时的具体原因；成功时显示 `-` |

### 6.6 当前批量导入边界

- 当前只支持单 SKU 批量计算，不支持在 Excel 中批量导入多 SKU 组合。
- 当前 Excel 文件只读取第一个工作表。
- 批量导入会使用页面当前车厢公差配置，但 Excel 模板本身不包含公差列。
- 当前结果弹窗支持状态筛选、失败原因筛选、差值筛选、只看需复核、导出当前筛选、导出需复核行和下载全部结果。
- 当前导入过程有 loading 遮罩、百分比进度、当前处理行提示和取消按钮。
- 当前代码中的人工码垛数量只校验“能转成数字”，没有强制正整数；从业务上建议后续收紧为非负整数。

## 7. 展示控制输入需求

展示控制不会改变装载计算结果，只影响 2D/3D 展示。

### 7.1 码垛进度

| 字段 | 代码字段 | 输入方式 | 默认值 | 当前约束 | 影响 |
| --- | --- | --- | ---: | --- | --- |

规则：

- 每次计算成功后，`visibleCount` 自动设置为最大装载量。
- 用户拖动进度条后，只改变可视化显示数量，不重新计算装载量。

| 端视图视角 | `角件端`、`柜门` | `角件端` | 控制端视图从柜内角件端看，还是从柜门端看 |
| 放大 2D 视图 | 当前视图 | 关闭 | 在弹窗中查看更大的 2D 画布 |

### 7.3 3D 视图输入

| 控制项 | 输入方式 | 影响 |
| --- | --- | --- |
| 旋转 | 鼠标 / 触控拖拽 | 调整 3D 观察角度 |
| 平移 | 鼠标 / 触控操作 | 调整 3D 画面位置 |
| 缩放 | 滚轮 / 触控缩放 | 调整远近 |
| 放大 3D 视图 | 点击放大按钮 | 在弹窗中查看更大的 3D 画布 |

## 8. 算法层入参结构

### 8.1 单 SKU 计算

单 SKU 计算核心入参：

```ts
calculatePacking(container, carton, options?)
```

其中：

```ts
interface ContainerSpec {
  id?: string;
  name?: string;
  length: number;
  width: number;
  height: number;
}

interface CartonSpec {
  length: number;
  width: number;
  height: number;
}

interface PackingOptions {
  cornerBlock?: CornerBlockSpec;
  strategy?: LoadingStrategy;
  clearance?: ContainerClearanceSpec;
}

interface ContainerClearanceSpec {
  front?: number;
  rear?: number;
  left?: number;
  right?: number;
  top?: number;
}
```

当前页面传入：

- `container`：来自柜型预设。
- `carton`：来自单 SKU 纸箱规格表单。
- `clearance`：来自车厢公差输入。
- `cornerBlock`：不从页面输入，算法使用默认角件。

### 8.2 多 SKU 计算

多 SKU 计算核心入参：

```ts
calculateMultiSkuPacking(container, skus, { strategy, clearance })
```

其中：

```ts
interface SkuInput extends CartonSpec {
  label: string;
  target: number;
  color: string;
}

type LoadingStrategy = "multi-destination" | "same-destination";
```

当前页面传入：

- `container`：来自柜型预设。
- `skus`：来自多 SKU 卡片列表，顺序即装载顺序。
- `strategy`：来自装载策略下拉框。
- `clearance`：来自车厢公差输入。

### 8.3 车厢公差派生结果

算法结果中会同时保留原始柜体和扣减后的有效柜体：

| 字段 | 说明 |
| --- | --- |
| `container` | 原始集装箱尺寸 |
| `effectiveContainer` | 扣减车厢公差后的有效装载尺寸 |
| `clearance` | 标准化后的 `front/rear/left/right/top` 公差 |

页面展示：

- 结果摘要显示有效尺寸。
- 本次排布说明显示是否启用车厢公差。
- 2D/3D 渲染会在启用公差时标注有效装载空间。

## 9. 状态与错误输入反馈

用户输入变化后，页面状态会用于提示当前结果是否仍有效：

| 状态 | 触发条件 | 用户含义 |
| --- | --- | --- |
| `待计算` | 初始状态，或还没有计算结果 | 需要点击计算 |
| `待重新计算` | 已有结果后修改了输入 | 当前可视化可能不是最新输入对应结果 |
| `已完成计算` | 计算成功且可装载数量大于 0 | 当前结果有效 |
| `无法装载` | 计算成功但数量为 0 | 当前尺寸无法装入选定柜型 |
| `计算失败` | 算法抛出错误 | 输入不合法或计算过程异常 |

典型错误：

- `纸箱长度必须为正数`
- `纸箱宽度必须为正数`
- `纸箱高度必须为正数`
- `至少选择一种纸箱朝向`
- `纸箱朝向不支持：xxx`
- `前公差必须为非负数`
- `公差扣减后的有效柜体长度必须为正数`
- `多 SKU 模式只支持 2 到 5 个 SKU`
- `A 目标数量必须为整数`
- `SKU 名称 "A" 不能重复`
- `柜型必须为 20GP、40GP、40HQ，或传入自定义柜体尺寸对象`

## 10. 当前未纳入输入的业务条件

以下条件当前没有输入入口，也不会参与计算：

| 业务条件 | 当前状态 | 后续建议 |
| --- | --- | --- |
| 纸箱重量 | 未输入 | 如用于机器人仿真或安全评估，应加入重量和承重约束 |
| 重心 | 未输入 | 后续可按 SKU / 箱体重量计算重心偏移 |
| 承重层数 | 未输入 | 可增加单箱最大承压、最大堆叠层数 |
| 纸箱朝向限制 | 页面支持 6 种朝向混合勾选 | 后续如进入机器人仿真，应继续补充抓取面、标签面、禁止倒置等语义 |
| 混装规则 | 多 SKU 只按策略启发式处理 | 需补充是否允许跨 SKU 填空、是否必须按客户分区 |
| 卸货顺序 | 仅通过 SKU 顺序和策略间接表达 | 可显式增加卸货地、优先级、装车顺序字段 |
| 纸箱公差 / 额外安全通道 | 未输入 | 车厢公差已支持；如需更精细控制，可增加纸箱公差、装卸通道、门端专项预留 |
| 角件尺寸 | 固定默认值 | 如果不同柜型角件不同，可开放为配置项 |
| 机器人夹具限制 | 未输入 | 后续仿真需要加入抓取面、夹具尺寸、避障空间 |
| 批量多 SKU | 未输入 | Excel 模板可扩展为多 SKU 明细表或多 sheet 模板 |

## 11. 后续输入需求建议

优先级建议如下：

| 优先级 | 建议输入项 | 原因 |
| --- | --- | --- |
| 高 | 人工码垛数量强制非负整数 | 当前批量导入只校验数字，业务语义需要更严格 |
| 高 | 明确 SKU 名称与装载顺序的关系 | 避免拖拽后用户误解 A/B/C 是商品名还是顺序号 |
| 高 | 多 SKU 是否允许跨 SKU 填空 | 这是异尺寸多 SKU 算法正确性的核心业务输入 |
| 中 | 自定义柜体尺寸 | 适配非标准柜或不同供应商内尺寸 |
| 中 | 车厢公差模板 / 按柜型保存 | 当前只保存一组本地公差；后续可按柜型或客户场景保存常用公差方案 |
| 中 | 批量导入朝向列 | 让 Excel 每行也能指定水平旋转、侧放或全自由朝向 |
| 中 | Excel 模板版本号 | 防止导入旧模板导致列语义混乱 |
| 低 | 纸箱重量、承重、重心 | 对最大装载量计算影响大，但需要真实业务数据支撑 |
