# 3D 端点标识优化设计

## 为什么做

当前 `柜门`、`角件端`、`X/Y/Z` 等标识主要依赖 WebGL sprite 文字。它们能跟随 3D 场景，但在缩小视图、弹框视图或高 DPI 屏幕上容易变糊，用户需要靠近看才能判断货柜方向。

本次优化的目标是让用户在首页 3D 卡片、放大弹框和坐标弹框中快速判断货柜方向，同时保持文字清晰和位置准确。

## 做什么

- 在 3D 货柜两端增加半透明颜色端面和细边框：
  - `柜门` 使用青绿色 `#42d6a4`，与现有 UI 中柜门色保持一致。
  - `角件端` 使用琥珀色 `#ffb24a`，避免纯红带来错误状态含义。
- 在 3D 容器右下角增加小图例：
  - 青绿色表示 `柜门`。
  - 琥珀色表示 `角件端`。
- 将关键文字标签改成 HTML/CSS 投影标签：
  - 标签的世界坐标仍来自 3D 场景模型。
  - 文字最终用 DOM 渲染，保证清晰度。
  - 覆盖首页卡片、放大弹框和坐标弹框。
- 保持箱体、货柜线框、角件块、坐标轴主体继续由 TresJS/Three.js 渲染。

## 不做什么

- 不给每一个箱子默认增加文字标签，避免大量 DOM 节点影响性能。
- 不改变装柜算法、坐标计算、装载顺序或 2D 视图规则。
- 不引入重型 3D 字体方案作为默认实现；如果未来需要大量 3D 文本，再评估 `troika-three-text`。

## 交互与视觉规则

- 端面颜色和细边框是主识别方式，文字标签是辅助确认。
- 图例固定在 3D 视图容器右下角，不随相机旋转。
- HTML 投影标签随相机、缩放和弹框尺寸变化而更新位置。
- 文字标签数量保持很少，仅用于 `柜门`、`角件端`、坐标轴字母和当前选中箱体。
- 标签不得遮挡主要货物视图；必要时允许端点标签在货柜外侧留出偏移。

## 技术设计

- 在 `src/renderers/cargoSceneModel.ts` 中扩展端点模型，输出端面/端框几何和投影标签锚点。
- 在 `Cargo3DSceneV2.vue` 中渲染端面/端框几何，并增加一个绝对定位的标签层和图例层。
- 标签层根据 Three.js camera 将 3D 世界坐标投影到屏幕坐标，使用 `transform: translate3d(...)` 更新位置。
- 所有 3D 入口继续复用 `Cargo3DSceneV2.vue`，避免首页卡片、放大弹框、坐标弹框出现三套标识逻辑。

## 验证方式

- 单元测试覆盖 `cargoSceneModel` 输出的端点颜色、几何位置和标签锚点。
- 源码结构测试确认三个 3D 入口都使用 `Cargo3DSceneV2.vue`。
- e2e 验证首页 3D、放大弹框、坐标弹框均能显示端点颜色和图例。
- 浏览器人工检查滚轮缩放、弹框缩放、标签清晰度和位置稳定性。
