# 装柜码垛术语约定

本文档用于统一算法、2D/3D 可视化、坐标表和测试中的码垛业务语言。后续涉及码垛顺序、标签展示、坐标导出时，优先以本文档为准。

## 项目展示坐标轴

以当前项目的 3D 坐标轴和“查看坐标”弹窗为准，用户可见坐标系定义为：

- 原点：角件端右下角。
- `X` 轴：柜宽方向，从原点所在侧壁向另一侧侧壁递增。
- `Y` 轴：柜长方向，从角件端、柜内最里面开始，向柜门方向递增。
- `Z` 轴：柜高方向，从地面向上递增。

因此：

- `Y = 0` 附近是角件端，也是码垛起始端。
- `Y` 越大，越靠近柜门端。
- `X = 0` 和 `X = 柜内宽度` 分别对应两侧侧壁。
- 同一装载步骤中，如果说“往前推进”，默认指沿 `Y` 轴从角件端推进到柜门端。

## 内部算法字段映射

当前算法内部的 `BoxPosition` 字段名和用户可见坐标轴不是同一套命名：

- 内部 `position.x`：柜长方向，对应用户可见 `Y` 轴。
- 内部 `position.y`：柜宽方向，对应用户可见 `X` 轴；坐标表会按“原点为角件端右下角，X 向左”做换算。
- 内部 `position.z`：柜高方向，对应用户可见 `Z` 轴。

后续讨论业务规则、3D 视角、坐标表时，优先使用用户可见坐标系；修改算法代码和测试断言时，需要明确说明使用的是内部字段。

## 核心术语

### 面

“面”是沿柜长方向 `Y` 切分出来的一个装载截面，也可以理解为从角件端到柜门端推进时的一段装载单元。

码垛顺序应按面推进：

```text
角件端 / Y=0
面 1 -> 面 2 -> 面 3 -> ... -> 柜门端
```

第一面必须从坐标轴 `Y=0` 附近开始；后续面按 `Y` 坐标递增顺序推进。

对应到内部算法字段，就是按 `position.x` 从小到大推进。

### 层

“层”是同一面内沿高度方向 `Z` 堆叠的一层。

每一面内部应按层堆叠：

```text
面 1：第 1 层 -> 第 2 层 -> ... -> 该面叠满
面 2：第 1 层 -> 第 2 层 -> ... -> 该面叠满
```

为避免歧义，本文档中的“层”只表示高度方向 `Z` 的层，不表示柜宽方向上的横向排布。

### 朝向

“朝向”描述同一面、同一层内纸箱的摆放方向，例如横放、竖放。

同一面、同一层内，可以同时包含横放和竖放纸箱。讨论码垛顺序时，只需要说明这些不同朝向的纸箱在当前面、当前层内全部摆放完毕，不再引入“排”“列”作为核心术语。

## 俯视统计口径

俯视图标签中的“横向占 29 列 / 竖向占 26 列 / 共 55 列”仅作为最终地面排布统计指标，不参与定义码垛顺序。

在 `40HQ / 488 x 380 x 291 mm / 单 SKU` 示例中，俯视图的柜长方向统计口径为：

- 横向占 `29` 列。
- 竖向占 `26` 列。
- 合计 `55` 列。

这里的 `55` 列是最终地面排布沿柜长方向的统计，不等同于码垛播放顺序中的“面数”，也不代表本文档核心术语中的“面”。在用户可见坐标系中，它对应沿 `Y` 方向推进的地面统计；在内部算法中，对应 `position.x` 方向的地面统计。

## 码垛顺序

标准码垛顺序规则为：

1. 起步约束：任意一面开始码垛时，第一箱必须紧贴集装箱侧壁，禁止从面中间位置起垛。
2. 面优先约束：码垛必须从角件端开始，沿用户可见 `Y` 轴向柜门方向推进，一面一面码放。当前面未叠满前，不允许开始下一面。
3. 分层约束：同一面内，必须从第 1 层开始逐层向上码放。同一层内，横放、竖放纸箱必须全部码放完毕后，才能开始该面的下一层。
4. 宽向压紧约束：同一端面投影内，横放、竖放纸箱应先互相靠紧；如果柜宽方向存在余量，余量必须落在外侧柜边，不允许出现在两组纸箱之间。
5. 排序约束：在最终排布已符合宽向压紧约束时，应保持最终排布、占用宽度、余量和最大装载量不变，仅调整纸箱的码垛先后顺序，使坐标表“序号”和 2D/3D 进度都按上述码垛顺序展示。

示意：

```text
面 1 第 1 层 -> 面 1 第 2 层 -> ... -> 面 1 叠满
面 2 第 1 层 -> 面 2 第 2 层 -> ... -> 面 2 叠满
面 3 第 1 层 -> 面 3 第 2 层 -> ... -> 面 3 叠满
...
柜门端最后一面
```

不应理解为：

```text
整柜第 1 层全部铺完 -> 整柜第 2 层全部铺完 -> ...
```

也不应理解为：

```text
某一个宽向位置一直堆高 -> 再堆下一个宽向位置
```

## 起步约束

任意一面开始码垛时，第一箱必须紧贴集装箱侧壁，禁止从宽向中间位置起垛。

判断口径：

- 如果第一箱贴近 `X = 0`，视为贴一侧侧壁。
- 如果第一箱贴近 `X = 柜内宽度`，视为贴另一侧侧壁。
- 如果第一箱既不贴 `X = 0`，也不贴 `X = 柜内宽度`，则视为从中间起垛，不符合规则。

对应到内部算法字段：

- 内部 `position.y = 0` 表示贴一侧侧壁。
- 内部 `position.y + position.dy = container.width` 表示贴另一侧侧壁。

角件避让可以影响具体坐标，但不应让某一面的第一箱从宽向中间开始。

混合横放、竖放时，不同朝向纸箱在柜长方向的长度可能不同。此时不能只按单个内部 `position.x` 分组判断“面”是否贴壁；应以端面投影的宽向覆盖为准：已装货物在宽向上应连续，内部不得出现断缝。

## 与装载结果的关系

码垛顺序是纸箱的先后顺序；最终装载结果是空间排布结果。正常情况下，只调整码垛顺序不应改变最终排布结果。

如果发现柜宽余量出现在横放组和竖放组之间，而不是出现在外侧柜边，则属于最终排布错误，需要先修正宽向位置，再讨论码垛顺序。

以 `40HQ / 488 x 380 x 291 mm / 单 SKU` 为基准，调整顺序时必须保持：

- 横向占 `29` 列。
- 竖向占 `26` 列。
- 合计 `55` 列。
- 占用宽度 `2332 mm`。
- 宽度余量 `20 mm`。
- 最大装载量 `1349` 箱。

该 `20 mm` 余量必须位于柜宽方向外侧边缘，不能位于横放组与竖放组之间。端视图在任意已形成端面的进度下，都不应把这 `20 mm` 显示为箱子之间的内部缝隙。

如果某次修改导致这些数值变化，说明修改影响了最终排布，而不只是调整码垛顺序，需要单独评估。

## 可视化和坐标表口径

- 3D 进度：应按码垛顺序显示已装箱数量。
- 2D 俯视图进度：应与 3D 使用同一套码垛顺序；可保留最终地面排布轮廓作为淡色参考。
- 坐标表：`序号` 应按码垛顺序递增；`装载顺序` 应与算法内部 `sequenceIndex + 1` 对齐。
- 坐标表中的位置字段用于定位纸箱位置，不应用来重新定义码垛先后顺序。

## 术语使用建议

为了减少歧义，后续讨论中建议使用下面说法：

- 使用“面”描述从角件端到柜门端的推进单元，也就是用户可见 `Y` 方向。
- 使用“层”描述高度方向 `Z` 的堆叠层。
- 使用“朝向”描述同一面、同一层内纸箱横放或竖放。
- 使用“码垛顺序”描述实际装箱先后。
- 使用“最终排布”描述装满后的空间布局和装载量。
